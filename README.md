# AI Chat Application

A browser-based AI chat and research application (v18) that runs as a fully static single-page application with optional cloud sync via Supabase. No server-side build step is required; all features are delivered through plain HTML, CSS, and JavaScript.

## Features

- **AI Chat** – Multi-turn conversations with support for multiple LLM models.
- **File Processing** – Handles PDFs, Word documents, Excel sheets, and images (OCR via Tesseract.js).
- **Canvas & Code Editor** – Collaborative editor with sandboxed iframe execution for multiple file types.
- **Terminal Integration** – WebSocket-based local shell bridge powered by xterm.js (requires the proxy server).
- **Research Tools** – Web search, deep research, image/video generation templates, and mind map visualization.
- **Chat Management** – Pin, folder, global search, and per-chat message search.
- **Document Generation** – Slide presentations, detailed reports, and PDF export via jsPDF.
- **Authentication** – GitHub OAuth, Google OAuth, and email + OTP via Supabase. Local-first with cloud sync.
- **Responsive UI** – Desktop (`index.html`) and mobile-optimised (`index-mobile.html`) variants with Tailwind CSS.

## Getting Started

### Prerequisites

- **Node.js ≥ 24** — required by the dev server and build scripts.

### Run locally

```bash
npm install
npm run dev
```

Open `http://localhost:3000/index.html` in your browser.

### Local terminal bridge (optional)

The Terminal tab inside the Canvas panel connects to a real local shell process. Start the proxy server in a separate terminal window:

```bash
npm run proxy
```

This starts both the LLM proxy and the xterm.js terminal bridge at `http://localhost:8787`. Requires `PROXY_UPSTREAM_API_KEY` or `OPENAI_API_KEY` to be set in your environment.

## Build

### Default (GitHub Pages / static hosting)

```bash
npm run build
```

Output is written to `dist/`. The build script copies all entry HTML files and their locally referenced assets.

### Puter platform

```bash
npm run build:puter
```

Output is written to `dist-puter/`. The modular HTML variant is promoted to `index.html` so that Puter serves it as the root page.

### Preview the production build

```bash
npm run dev:dist
```

Serves the `dist/` folder on port 4173.

### Extract inline CSS/JS (modular variant)

```bash
npm run split:html
```

Extracts large inline `<style>` and `<script>` blocks from the all-in-one HTML files into `assets/css/` and `assets/js/`, producing the `*.modular.html` variants.

### Run all checks

```bash
npm run check
```

Runs tests and then builds.

## Testing

```bash
npm test
```

Executes `tests/smoke.test.mjs`, which verifies:

1. Entry HTML files exist and contain no stale `manifest.json` references.
2. Security helpers `sanitizeHtml()` and `renderMarkdownHtml()` are present.
3. The canvas `<iframe>` uses `sandbox="allow-scripts allow-downloads"`.
4. The build script produces `dist/` entry files.
5. The proxy script exists.
6. Modular HTML files reference external CSS/JS assets correctly.
7. The Puter build creates `dist-puter/index.html` from the modular variant.
8. The deploy console (`puter-deploy.html`) uses external asset files.

> **Windows note:** The smoke test can fail with `EBUSY` if a file in `dist/` is locked by another process.

## Database Schema (Supabase)

| Table | Key Columns | Notes |
|---|---|---|
| `user_profiles` | `id` (uuid) · `email` · `full_name` · `avatar_url` | Auto-created on signup; user-scoped RLS |
| `chats` | `id` (bigint) · `user_id` · `title` · `messages` (jsonb) · `pinned` · `folder` · `canvas` (jsonb) | Indexed on `(user_id, updated_at DESC)` |
| `shared_chats` | — | Public chat sharing |

Row-Level Security is enabled on all tables so users can only access their own data.

## Deployment

CI/CD is handled by `.github/workflows/static.yml`. On every push to `main`:

1. Install dependencies (`npm ci`).
2. Run smoke tests (`npm test`).
3. Build the static site (`npm run build`).
4. Deploy `dist/` to **GitHub Pages** automatically.

## Environment Variables

| Variable | Used by | Description |
|---|---|---|
| `PROXY_UPSTREAM_API_KEY` | `proxy.mjs` | API key forwarded to the upstream LLM |
| `OPENAI_API_KEY` | `proxy.mjs` | Fallback OpenAI-compatible API key |
| `PORT` | `dev.mjs` | Override the dev server port (default `3000`) |

Keep secrets out of version control; add them to your shell profile or a `.env` file (already listed in `.gitignore`).
